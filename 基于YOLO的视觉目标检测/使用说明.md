## 三、标注软件安装与数据标注

### 3.1 标注软件选择

| 工具 | 类型 | 适用场景 | 安装方式 |
|------|------|----------|----------|
| **LabelImg** | 桌面端 | 小规模标注（推荐入门） | pip 一键安装 |
| **CVAT** | Web 端 | 团队协作、大规模标注 | Docker 部署 |
| **Labelme** | 桌面端 | 多边形标注（不规则缺陷） | pip 安装 |
| **Roboflow** | 在线平台 | 自动标注 + 数据增强 | 网页端注册 |

**推荐方案**：少量图片用 LabelImg，大量图片用 CVAT。

### 3.2 LabelImg 安装与使用

#### 安装

```bash
conda activate yolov11
pip install labelImg
```

#### 启动

```bash
labelImg
```

#### 标注步骤

1. **打开图片目录**：点击左上角 `Open Dir` → 选择存放图片的文件夹
2. **设置标注格式**：点击顶部 `View` → `Change Save Dir` → 选择保存目录；点击 `YOLO` 按钮切换为 YOLO 格式（默认是 PascalVOC）
3. **创建标注框**：点击 `Create RectBox` → 在图片上拖拽框选目标
4. **填写类别**：弹出对话框中输入类别名（如 `scratch`），点击 OK
5. **保存**：按 `Ctrl+S` 保存（或勾选 `Auto SaveMode` 自动保存）
6. **下一张**：按 `D` 下一张 / `A` 上一张

> **快捷键**：`W` 创建框 / `D` 下一张 / `A` 上一张 / `Ctrl+S` 保存 / `Del` 删除选中标注

#### 标注产出

每张图片会生成一个同名 `.txt` 文件：

```
# image_001.txt
0 0.450000 0.320000 0.120000 0.080000
```

格式为 `class_id x_center y_center width height`（归一化坐标，0~1）。

### 3.3 CVAT 安装与使用（大规模标注推荐）

#### Docker 部署

```bash
# 克隆仓库
git clone https://github.com/cvat-ai/cvat.git
cd cvat

# 启动服务
docker compose up -d

# 创建管理员账号
docker exec -it cvat_server bash -ic 'python3 ~/manage.py createsuperuser'
```

浏览器打开 http://localhost:8080，登录后：

1. **创建项目**：Projects → Create → 填写名称、标签（类别名）
2. **创建任务**：Tasks → Create → 上传图片集 → 关联项目
3. **开始标注**：点击任务 → Open → 框选目标 → 填写标签
4. **导出数据**：Tasks → 选择任务 → Export → 格式选 `YOLO 1.1` → 下载 zip

#### 导出格式转换

CVAT 导出的 YOLO 格式可直接被 `prepare_dataset.py` 识别：

```bash
cd backend
python prepare_dataset.py --source ./cvat_export --classes scratch dent crack
```

### 3.4 标注规范与注意事项

| 规则 | 说明 |
|------|------|
| **标注框贴合** | 边框尽量紧贴目标边缘，不要留太多空白 |
| **不遗漏** | 图片中所有目标都要标注，即使是部分遮挡的 |
| **类别一致** | 同类目标使用相同的类别名，不要出现 "划痕" 和 "scratch" 混用 |
| **小目标** | YOLO 对极小目标（< 8px）检测效果差，可放大图片后标注 |
| **图片质量** | 模糊、过曝、过暗的图片应剔除，不要标注 |
| **数量均衡** | 各类别样本数量尽量接近，差异不超过 3 倍 |

### 3.5 数据增强建议（标注后）

原始图片数量不足时，可通过数据增强扩充：

| 方法 | 工具 | 说明 |
|------|------|------|
| 翻转/旋转/裁剪 | LabelImg 自带 / Albumentations | 几何变换 |
| 亮度/对比度/噪声 | Albumentations | 光照模拟 |
| Mosaic | YOLO 训练自带 | 4 张图拼接，提升小目标检测 |
| MixUp | YOLO 训练自带 | 2 张图透明叠加 |
| Copy-Paste | SAHI / Roboflow | 将目标复制粘贴到新背景 |

> 本项目的 `train.py` 已内置 Mosaic、MixUp、HSV 增强、翻转、旋转、缩放等数据增强，无需额外处理。

## 四、数据集准备

### 4.1 数据集目录结构

```
backend/dataset/
├── images/
│   ├── train/        # 训练图片 (.jpg / .png)
│   └── val/          # 验证图片
├── labels/
│   ├── train/        # 训练标注 (.txt)
│   └── val/          # 验证标注
```

### 4.2 YOLO 标注格式

每个图片对应一个同名 `.txt` 文件，每行一个目标：

```
class_id  x_center  y_center  width  height
```

- `class_id`：类别索引，从 0 开始（与 `data.yaml` 中 `names` 的顺序对应）
- `x_center, y_center`：中心点归一化坐标（0~1）
- `width, height`：宽高归一化值（0~1）

示例：

```
0 0.450000 0.320000 0.120000 0.080000
2 0.780000 0.650000 0.050000 0.150000
```

### 4.3 自动构建数据集（推荐）

```bash
cd backend

# YOLO 格式标注
python prepare_dataset.py --source ./原始数据目录 --classes scratch dent crack

# VOC XML 格式标注（LabelMe / LabelImg 默认格式）
python prepare_dataset.py --source ./原始数据目录 --format voc --classes scratch dent crack

# COCO JSON 格式标注（CVAT / Roboflow 导出格式）
python prepare_dataset.py --source ./原始数据目录 --format coco --classes scratch dent crack
```

脚本会自动完成：标注校验 → 按 80/20 划分 train/val → 生成 `data.yaml`。

### 4.4 修改类别配置

编辑 `backend/data.yaml`：

```yaml
path: ./dataset
train: images/train
val: images/val

nc: 3                              # 类别数
names:
  0: scratch                       # 划痕
  1: dent                          # 凹陷
  2: crack                         # 裂纹
```

### 4.5 校验数据集

```bash
python prepare_dataset.py --validate
```

## 五、训练

### 5.1 迁移学习原理

YOLOv11 使用 **迁移学习（Transfer Learning）** 策略：

```
COCO 预训练权重 (yolo11n.pt, 80类通用物体)
        │
        ▼  替换最后的检测头，用你的数据微调
自定义缺陷检测模型 (best.pt, N类缺陷)
```

- **冻结层**：预训练模型的前几层（提取边缘、纹理等通用特征）在训练初期保持不变
- **微调层**：后几层（提取高级语义特征）和检测头全程参与训练
- **优势**：少量数据（几百张）即可获得不错效果，远优于从零训练

### 5.2 默认训练

```bash
cd backend
python train.py
```

默认参数：`yolo11n.pt` 预训练权重，100 epochs，`imgsz=640`，`batch=16`。

### 5.3 自定义训练

```bash
# 更大模型 + 更大输入尺寸（精细缺陷推荐）
python train.py --model yolo11s.pt --epochs 200 --imgsz 1280 --batch 8

# 小数据集（<500张）建议降低学习率
python train.py --lr0 0.001 --epochs 200

# 快速验证流程是否跑通（5 epochs）
python train.py --quick

# 从零训练（不用预训练权重，需要大量数据）
python train.py --no-pretrained --epochs 500
```

### 5.4 模型选择参考

| 模型 | 参数量 | 输入尺寸 | 速度 | 精度 | 显存需求 | 适用场景 |
|------|--------|----------|------|------|----------|----------|
| yolo11n.pt | 2.6M | 640 | 最快 | 一般 | ~2GB | 快速原型验证 |
| yolo11s.pt | 9.4M | 640 | 快 | 较好 | ~4GB | **推荐起步** |
| yolo11m.pt | 20.1M | 640 | 中等 | 好 | ~6GB | 中等规模数据 |
| yolo11l.pt | 25.3M | 640 | 较慢 | 很好 | ~8GB | 大规模数据 |
| yolo11x.pt | 56.9M | 640 | 最慢 | 最好 | ~12GB | 追求极限精度 |

> 显存不足时可减小 `--batch`（如 `--batch 4`）或 `--imgsz`（如 `--imgsz 416`）。

### 5.5 输出文件

训练产物位于 `backend/runs/train/defect_detect/`：

```
runs/train/defect_detect/
├── weights/
│   ├── best.pt            # 最佳权重（部署用这个）
│   └── last.pt            # 最终权重（断点续训用）
├── results.csv            # 训练指标数据
├── confusion_matrix.png   # 混淆矩阵
├── F1_curve.png           # F1 曲线
├── PR_curve.png           # 精确率-召回率曲线
├── P_curve.png            # 精确率曲线
├── R_curve.png            # 召回率曲线
└── results.png            # 训练损失/指标曲线图
```

### 5.6 训练效果评估

| 指标 | 含义 | 健康范围 |
|------|------|----------|
| **mAP@50** | IoU=0.5 时的平均精度 | > 0.7 为可用 |
| **mAP@50:95** | IoU=0.5~0.95 的平均精度 | > 0.5 为良好 |
| **Precision** | 精确率（检测结果中多少是对的） | > 0.8 |
| **Recall** | 召回率（真实目标中检出了多少） | > 0.7 |
| **train/box_loss** | 边框回归损失 | 持续下降 |
| **train/cls_loss** | 分类损失 | 持续下降 |

> 若 `val/box_loss` 上升而 `train/box_loss` 下降 → 过拟合，增加数据或减小模型。

### 5.7 断点续训

训练中断后从 `last.pt` 继续：

```bash
python train.py --model runs/train/defect_detect/weights/last.pt --epochs 200
```

## 七、训练超参速查

| 参数 | 默认值 | 说明 | 调参建议 |
|------|--------|------|----------|
| `--epochs` | 100 | 训练轮次 | 数据少 → 200~300 |
| `--batch` | 16 | 批次大小 | 显存不足 → 8 或 4 |
| `--imgsz` | 640 | 输入尺寸 | 精细缺陷 → 1280 |
| `--lr0` | 0.01 | 初始学习率 | 小数据集 → 0.001 |
| `--patience` | 50 | 早停轮数 | 0 = 关闭早停 |
| `--mosaic` | 1.0 | Mosaic 增强 | 小目标多 → 1.0 |
| `--close-mosaic` | 10 | 最后 N 轮关闭 Mosaic | 一般 10~20 |
| `--model` | yolo11n.pt | 预训练模型 | n/s/m/l/x 按显存选 |
| `--optimizer` | auto | 优化器 | auto 自动选择 |
| `--weight_decay` | 0.0005 | L2 正则化 | 过拟合 → 0.001 |
